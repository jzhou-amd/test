## What We Learned

### Architecture Understanding

**Data Flow**:
```
AgentBench → Dynamo Frontend (HTTP :8000) → SGLang Worker (GPU) → Response
                    ↓                              ↓
              etcd (:2379)                   NATS (:4222)
              (service registry)              (message broker)
```

**Docker Image Types**:
1. **Published (Smoke Test)**: `nvcr.io/nvidia/ai-dynamo/sglang-runtime:1.0.2`
   - Pre-built by NVIDIA
   - No instrumentation
   - Used for validation only

2. **Instrumented (Custom Build)**: `local/dynamo-frontend:runtime-json-logs`, `local/dynamo-sglang:runtime-json-logs`
   - Built from patched source
   - Includes "blue code" (hint preservation + JSON logging)
   - Required for hint proof-of-concept

**Model Caching**:
- Host path: `/mnt/docker-data/dynamo_cache/`
- Container mount: `-v /mnt/docker-data/dynamo_cache:/models/hfcache`
- Shared across all users (777 permissions)
- Model: `Qwen/Qwen2.5-7B-Instruct` (~14GB, cached for reuse)
